## 身份与凭据(一次性)

- **Publisher** 已在 https://marketplace.visualstudio.com/manage 注册,ID = `patrick1099`,
  必须与 `package.json` 的 `"publisher"` 字段一致。
- **PAT(Personal Access Token)** 在 https://dev.azure.com 生成,
  Scope = **Marketplace → Manage**,Organization = **All accessible organizations**。
  - 默认有效期通常一年,到期后 CI 发布会失败,届时重新生成并更新 GitHub secret。
  - 校验是否有效:`npx @vscode/vsce verify-pat patrick1099`(读环境变量 `VSCE_PAT`)。

## GitHub secret(自动发布前置)

仓库 → Settings → Secrets and variables → Actions → New repository secret
- Name:`VSCE_PAT`
- Value:PAT 字符串

`.github/workflows/publish.yml` 引用 `${{ secrets.VSCE_PAT }}`。换 PAT 后必须同步更新这里。

## 日常发新版(走自动发布)

1. 改 `package.json` 的 `version`(如 `0.1.0` → `0.1.1`),按需更新 `CHANGELOG.md`。
2. 提交并打标签:
   ```bash
   git commit -am "release: v0.1.1"
   git tag v0.1.1
   git push --follow-tags
   ```
3. 推送 `v*` 标签触发 `.github/workflows/publish.yml`:
   `npm ci` → `npm run test:unit` → `vsce publish --no-dependencies`,
   发布的是 **package.json 里当前的 version**(不是标签名,二者保持一致即可)。
4. 几分钟后市场页面更新。

> 版本号一旦发布即"烧掉",撤回(unpublish)后也不能再用同一号,只能升号重发。

## 手动发布(应急 / 首次)

CI 不可用时本地直接发(首版 v0.1.0 即这样发的):
```bash
# 先验证打包无误
npx @vscode/vsce package --no-dependencies -o code-jump-tags.vsix
# 用 PAT 发布(VSCE_PAT 走环境变量,勿写进任何提交文件)
VSCE_PAT='<PAT>' npx @vscode/vsce publish --no-dependencies
```

---

## 注意事项

- `*.vsix` 已被 `.gitignore` 忽略,不会进仓库。
- PAT 绝不写进任何会提交的文件;只用环境变量或 GitHub secret。
- `package.json` 当前 `"preview": true`,市场会标 "Preview";正式稳定后可去掉。
- `--no-dependencies`:扩展用 webpack 打包,运行时依赖已 bundle,无需让 vsce 再打包 node_modules。
