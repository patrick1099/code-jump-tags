# 0.7.2 发版 workflow 升级 actions v5（清除 Node 20 弃用注解）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **背景：** 0.7.0 发版的 publish.yml CI 跑成功，但带一条**非致命**注解：「Node.js 20 is deprecated. actions/checkout@v4、actions/setup-node@v4 target Node 20 but are being forced to run on Node 24」。GitHub 将停用 Node 20 runner（见 github.blog 2025-09-19 公告）。把这两个 action 升到 v5（其 runtime 为 Node 24）即可清除注解。这是纯 CI/构建维护，不改任何扩展功能代码。

> **关键时序：** `publish.yml` 只在推送 `v*` tag 时运行。所以本计划对 workflow 的改动**要等推送 `v0.7.2` tag 时才被实际执行**——即 0.7.2 自身的发版 run 就是对「新 workflow（v5）能否干净发布、注解是否消失」的验证。

**Goal:** 把 `.github/workflows/publish.yml` 的 `actions/checkout` 与 `actions/setup-node` 从 v4 升到 v5，作为 0.7.2 维护版发布，发版时确认 Node 20 弃用注解消失。

**Architecture:** 单文件改两行 tag（`@v4`→`@v5`），随附版本号 0.7.1→0.7.2 与 CHANGELOG。0.7.2 的扩展功能代码与 0.7.1 完全一致——本版唯一实质变化是发版流水线。发布 v0.7.2 既走通新流水线、又把维护说明落到市场版本号上。

**Tech Stack:** GitHub Actions（YAML）、`@vscode/vsce`、webpack、vitest。

**当前 `publish.yml` 的 `steps` 段（供定位）：**
```yaml
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "18"
          cache: "npm"
      - run: npm ci
      - run: npm run test:unit
      - name: Publish
        run: npx @vscode/vsce publish --no-dependencies
        env:
          VSCE_PAT: ${{ secrets.VSCE_PAT }}
```
> `setup-node@v5` 仍支持 `node-version` 与 `cache` 输入，无破坏性变更；`node-version: "18"`（构建用的工具链 node，与被弃用的 action runtime node 是两码事）**保持不动**，不在本期扩大范围。
